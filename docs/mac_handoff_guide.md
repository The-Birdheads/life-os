# Mac代行者向け：Habitas ビルド＆アップロード手順書

この手順書は、Habitas（ハビタス）のソースコードから iOS アプリをビルドし、App Store Connect へアップロードするための手順をまとめたものです。

## 1. 事前準備 (代行者のPC環境)
以下のツールがインストールされている必要があります。
- **Node.js** (v18以上推奨)
- **CocoaPods** (`sudo gem install cocoapods`)
- **Xcode** (最新版)

## 2. セットアップ
リポジトリをクローンし、必要なライブラリをインストールします。

```bash
# プロジェクトのディレクトリに移動
cd life-os

# ライブラリのインストール
npm install

# Web版のビルド
npm run build

# Capacitor の同期 (iOS用の実行ファイル生成)
npx cap sync ios
```

## 3. Xcode での作業
1.  プロジェクトを開く
    ```bash
    npx cap open ios
    ```
2.  **Signing (署名) の設定**
    - 左側のプロジェクトナビゲーターで「App」を選択。
    - 「Signing & Capabilities」タブを開く。
    - 「Team」にて、依頼主（開発者アカウント）のチームを選択。
3.  **アーカイブの作成**
    - Xcode上部の実行ターゲットを **「Any iOS Device (arm64)」** に変更。
    - メニューから **Product > Archive** を実行。
4.  **アップロード**
    - アーカイブ完了後に開くウィンドウ（Organizer）で、該当のアーカイブを選択。
    - **Distribute App** をクリック。
    - **Custom** > **App Store Connect** > **Upload** の順に選択し、画面の指示に従い完了させる。

## 4. スクリーンショットの撮影
App Store 掲載用に、以下のシミュレータでスクリーンショットを撮影してください。

### 対象デバイス（重要：この2つのサイズは必須です）
- **iPhone 14/15 Pro Max** (6.5インチ相当)
- **iPhone 8 Plus** (5.5インチ相当)

### 撮影の手順
1. Xcode上部のターゲットで上記のデバイスを選択し、`Command + R` で実行します。
2. アプリが起動したら、主要な画面を表示します。
3. `Command + S` でスクリーンショットがデスクトップに保存されます。

### 撮影してほしい画面（各4枚）
- **前提**: 日付を2026-03-09に設定
- **記録タブ/すべて/メイン画面**: メインの進捗画面。
- **記録タブ/すべて/+ボタン/タスクの登録**: タスクの追加画面。
- **振り返りタブ**: 振り返りメモやダッシュボード、完了済みの確認。
- **週タブ**: 1週間の振り返り画面。

## 5. 完了後の連絡
アップロードが正常に完了し、スクリーンショットが撮れたら、それらを依頼主へ送付または共有してください。
