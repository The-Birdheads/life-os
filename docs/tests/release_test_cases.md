# Habitas (LifeOS) リリース前テストケース

アプリのリリース前にシミュレータまたは実機で確認すべき、手動テストケース（機能試験項目）の一覧です。

## 1. 認証機能 (Auth)
| ID | テスト項目 | 手順 | 期待される結果 | 備考/ステータス |
| :---: | :--- | :--- | :--- | :--- |
| **AUTH-01** | 初回起動とローカルDB生成 | 新規インストールしたアプリを起動する | 初期化画面が表示され、その後「Today」画面に遷移する（ローカルユーザーとして初期化される） | |
| **AUTH-02** | Googleログイン | 設定等にある「Googleでログイン」ボタンをタップする | ブラウザまたはSafariVCが開き、Google認証後、アプリにリダイレクトしてログイン状態になる | AppUrlOpen(Capacitor)の連携確認 |
| **AUTH-03** | ログアウト時の動作 | ログイン状態で「ログアウト」を実行する | クラウドセッションが破棄され、ローカルユーザーに切り替わる。または初期画面に戻る | |

## 2. データ管理 / クラウド同期 (Sync)
| ID | テスト項目 | 手順 | 期待される結果 | 備考/ステータス |
| :---: | :--- | :--- | :--- | :--- |
| **SYNC-01** | オフラインデータのクラウド移行 | 未ログイン状態（ローカル）でタスクや行動を記録したのち、Googleログインを行う | ローカルで作成したデータがログインしたアカウントに移行・同期され、引き続き表示される | |
| **SYNC-02** | 複数端末間の同期確認 | 端末Aでタスクを追加し、端末Bで同じアカウントでログイン/起動する | 端末Bで同期処理が走り、端末Aで追加したタスクが正常に表示される | |
| **SYNC-03** | オフライン状態での動作 | ネットワークを切断（機内モード）した状態でタスク追加や完了操作を行う | エラーにならず、ローカルのSQLiteに正常に保存され、一覧にも反映される | |

## 3. タスク・行動管理 (CRUD / Check)
| ID | テスト項目 | 手順 | 期待される結果 | 備考/ステータス |
| :---: | :--- | :--- | :--- | :--- |
| **TASK-01** | 習慣 (Habit) の追加と表示 | 「+」ボタン（FAB）から新規習慣を追加する（タイトル、優先度、Vol指定） | TodayViewの「習慣」セクション、または「すべて」セクションに作成した習慣が表示される | |
| **TASK-02** | 習慣の完了チェック | 習慣のチェックボックスをタップする | チェックが入り、ステータスが「完了」として保存される（見た目が沈み込み等の変化をする） | |
| **TASK-03** | タスク (Oneoff) の追加と編集 | 新規タスク（期限付き）を追加し、その後アイテム部分をタップして編集ダイアログから内容を変更・保存する | 内容が正しく更新される（期限の変更も反映される） | |
| **TASK-04** | 行動 (Action) の記録と削除 | 行動ログを記録する。その後アイテムをタップし、ダイアログ下部から「削除」を実行する | アイテムが正常に削除され、一覧から消える | |

## 4. ダッシュボード表示 (TodayView)
| ID | テスト項目 | 手順 | 期待される結果 | 備考/ステータス |
| :---: | :--- | :--- | :--- | :--- |
| **DASH-01** | タブ（フィルタ）の切り替え | Today画面で「すべて」「習慣」「タスク」「行動」のフィルタを切り替える | 選択したカテゴリのデータのみが画面に表示され、他は非表示になる | |
| **DASH-02** | 完了したタスクのソート | 複数のタスクがある状態で、一部を完了にする | （設定したソート順に基づいて）完了済みアイテムがリスト上で適切に並び替えられる | |

## 5. 振り返り機能 (ReviewView)
| ID | テスト項目 | 手順 | 期待される結果 | 備考/ステータス |
| :---: | :--- | :--- | :--- | :--- |
| **REV-01** | 充実度とメモの入力・保存 | Review画面を開き、充実度スライダーを動かし、メモを入力して「保存」を押す | 保存が完了し、画面が「プレビュー（登録済み確認）」モードに切り替わる | |
| **REV-02** | 保存済み振り返りの編集 | 保存された振り返り画面右上（または領域）の「編集」ボタン（鉛筆アイコン）を押す | 再度入力フォームが表示され、既存の値がセットされた状態で編集できる | |
| **REV-03** | サマリーの正確性 | Today画面で習慣1つ、タスク2つを完了させてからReview画面を開く | ダッシュボードセクションで、習慣=1件、タスク=2件と正しく集計されている | |

## 7. 設定・通知・その他
| ID | テスト項目 | 手順 | 期待される結果 | 備考/ステータス |
| :---: | :--- | :--- | :--- | :--- |
| **ETC-01** | 通知スケジュールの登録 | 通知設定モーダルを開き、通知する時間を設定・保存する（iOSでは権限プロンプトが出る） | 権限が許可されたのち、指定時間にローカル通知が飛ぶこと | 実機での確認を推奨 |
| **ETC-02** | 広告の表示位置確認 | 各主要画面を遷移して下部にスクロールする | バナー広告（テスト広告）の表示領域（AdHeight）分、コンテンツが上に押し上げられ被らない | |

---

### テスト実施方法
*   「状態/備考」欄に `[ ] 未実施`, `[x] パス`, `[!] 失敗/バグあり` 等のステータスを記入して検証にお使いください。
*   iOSリリースに向けては特に **AUTH-02 (OAuthログイン)** や **ETC-01 (通知権限・ローカル通知)** がiOS固有のネイティブ機能（Capacitorプラグイン）に依存するため、入念な確認をお勧めします。
